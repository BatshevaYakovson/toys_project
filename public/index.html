API Documentation

User API

Base URL

/api/users

Endpoints

1. Get Users

GET /

Placeholder endpoint to verify the user route is functioning.

2. Get User Information

GET /myInfo

Retrieves the information of the authenticated user.

Authentication
Requires user authentication.

3. Get Users List

GET /usersList

Retrieves a list of all users (Admin only).

Authentication
Requires admin authentication.

4. Register New User

POST /

Registers a new user.

Request Body 
{
  "name": "User Name",
  "email": "user@example.com",
  "password": "password123"
}

5. User Login

POST /login

Logs in an existing user.

Request Body Response 
{
  "email": "user@example.com",
  "password": "password123"
}
        
{
  "token": "JWT_TOKEN"
}

Toys API

Base URL

/api/toys

Endpoints

1. Get List of Toys

GET /

Retrieves a paginated list of toys.

Parameter Description
perPage (optional) Number of items per page (default: 5)
page (optional) Page number (default: 1)

2. Search for Toys

GET /search

Searches for toys based on a specified query string.

Parameter Description
s Query string for toy name search.

3. Add a New Toy

POST /

Adds a new toy to the database.

Authentication Request Body
Requires user authentication.
